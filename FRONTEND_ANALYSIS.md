## 项目概述

### 基本信息

| 属性 | 值 |
|------|-----|
| 项目名称 | CBoard V2 |
| 类型 | 代理订阅聚合管理平台 |
| 前端框架 | Vue 3 + TypeScript |
| 用途 | 用户订阅管理、节点聚合、费用结算 |
| 主要应用者 | 最终用户 + 平台管理员 |

### 核心功能

- ✅ 用户认证（邮箱/密码、Telegram）
- ✅ 订阅管理（设备管理、Clash URL 导出）
- ✅ 订单系统（多套餐、支付方式多样）
- ✅ 工单系统（用户-管理员沟通）
- ✅ 邀请系统（佣金返利）
- ✅ 盲盒系统（积分消耗、随机奖励）
- ✅ 完整管理后台（用户、订单、套餐、节点等）

## 用户端页面详解

### 1. 🎯 Dashboard（仪表盘）

**页面路由：** `/`  
**组件：** `dashboard/Index.vue`

**主要功能和按钮：**

| 组件/功能 | 按钮名称 | 功能说明 |
|----------|---------|---------|
| **欢迎卡片** | - | 显示用户名、等级徽章、是否有折扣 |
| **账户余额卡** | `充值` | 跳转到 `/recharge` |
| **签到卡** | `签到` | 调用 checkIn()，获得签到奖励 |
| **订阅卡** | `升级套餐` | 打开升级模态框 |
| **订阅卡** | `更多` | 跳转到 `/subscription` |
| **工单提醒** | `查看工单` | 跳转到 `/tickets` |
| **公告区** | `了解更多` | 显示完整公告详情 |

**显示的数据和统计：**
- 用户等级与折扣率
- 账户余额
- 连续签到天数
- 订阅剩余天数
- 设备使用情况 vs 设备限制
- 流量使用情况
- 最近 5 条活动
- 未读工单数
- 最新公开公告

## 认证流程

### 用户登录流程

```
1. 用户输入邮箱和密码
   ↓
2. 点击"登录"按钮
   ↓
3. 调用 userStore.login(email, password)
   ↓
4. 后端返回 {access_token, refresh_token}
   ↓
5. 客户端保存到 localStorage
   ↓
6. 自动调用 fetchUser() 获取用户信息
   ↓
7. 判断用户是否为 admin
   ↓
8. 重定向到 /admin (如果 admin) 或 / (用户首页)
```

### 路由守卫（router/index.ts）

```typescript
// 全局前置守卫
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  // Guest 路由：已登录者重定向
  if (to.meta.guest && userStore.isLoggedIn) {
    next('/')
  }
  
  // Auth 路由：未登录者重定向
  else if (to.meta.auth && !userStore.isLoggedIn) {
    next('/login')
  }
  
  // Admin 路由：无权限者重定向
  else if (to.meta.admin && !userStore.isAdmin) {
    next('/')
  }
  
  else {
    next()
  }
})
```

### Token 管理

- **自动注入：** 每个 HTTP 请求自动在 `Authorization` 头添加 `Bearer {token}`
- **刷新机制：** Token 过期时自动使用 `refresh_token` 获取新 token
- **登出清理：** 登出时清除所有本地存储的 token
