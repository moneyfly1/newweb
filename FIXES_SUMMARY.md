# 用户端修复总结

## 修复时间
2026-03-03

## 修复的问题

### 1. ✅ 订单列表支付方式显示问题
**问题**: 订单列表中"支付方式"列显示为 "-"

**原因**: 前端使用 `payment_method` 字段，但后端返回的是 `payment_method_name`

**修复**:
- 修改 `Order/Index.vue` 第 294 行和第 159 行
- 将 `payment_method` 改为 `payment_method_name`

**影响**: 订单列表和详情页现在能正确显示支付方式（支付宝、微信支付等）

---

### 2. ✅ 用户端弹窗改为抽屉形式
**问题**: 用户端所有操作（购买、支付、编辑、详情等）都是居中弹窗（Modal），不符合现代 UI 设计

**修复**: 将所有用户端页面的 Modal 改为 Drawer（右侧抽屉）

**修改的页面**:
1. **Shop.vue** - 套餐购买
   - 购买确认抽屉
   - 扫码支付抽屉
   - 加密货币支付抽屉

2. **Order/Index.vue** - 订单列表
   - 支付抽屉
   - 订单详情抽屉
   - 扫码支付抽屉
   - （已经使用 Drawer，无需修改）

3. **Subscription/Index.vue** - 订阅管理
   - 二维码抽屉
   - 重置订阅抽屉
   - 转换余额抽屉
   - 升级支付抽屉
   - 支付二维码抽屉
   - 加密货币支付抽屉
   - 升级订阅抽屉

4. **Recharge/Index.vue** - 充值页面
   - 扫码支付抽屉

5. **Invite/Index.vue** - 邀请码
   - 创建邀请码抽屉

6. **Ticket/Index.vue** - 工单
   - 创建工单抽屉

7. **Device/Index.vue** - 设备管理
   - 删除确认抽屉

**技术细节**:
- 使用 `CommonDrawer` 组件替代 `n-modal`
- 统一使用 `@confirm/@cancel` 事件处理
- 调整宽度属性：`:width="500"` 替代 `style="width: 500px;"`
- 使用 `show-footer` 和 `:loading` 属性

**用户体验提升**:
- ✅ 统一的右侧抽屉交互
- ✅ 不遮挡主要内容，可以看到背景
- ✅ 更流畅的滑入/滑出动画
- ✅ 更好的移动端适配
- ✅ 符合现代 Web 应用设计规范
